BUILDING

payment-processing-platform

The four microservices above represent a single bank — one ledger, one set of accounts, one institution (think Barclays). This platform sits above multiple independent instances of that ecosystem, each acting as a separate bank (Barclays, Lloyds, HSBC), and orchestrates payments between them. It does not own any accounts or balances — it coordinates money movement across institutions, handles distributed transaction failures with a compensation chain, guarantees exactly-once processing, and notifies merchants of payment outcomes via webhooks.

Java 21 Spring Boot 3 Spring Cloud Gateway Apache Kafka PostgreSQL Redis Saga Pattern Outbox Pattern Event Sourcing Resilience4j Prometheus Docker
